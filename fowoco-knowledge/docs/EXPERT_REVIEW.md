# 전문가 검수 운영 기준

## 왜 필요한가

팀원 두 명의 독립 검수는 **라벨 기준의 일관성**을 확인합니다.
그러나 다음 내용이 실제 제도·현장과 맞는지는 도메인 경험 없이 판단할 수 없습니다.

- E-9 체류·계약·고용변동 Workflow 순서
- 실제로 요청하는 서류명과 조건
- 급여·공제·퇴사 안내의 위험표현
- 외국인근로자가 이해하기 어려운 한국어 표현

따라서 팀 검수와 전문가 검수는 서로 대체하지 않고 목적을 분리합니다.

## 전문가 역할

| 검수 영역 | 적합한 검수자 | 확인 내용 |
| --- | --- | --- |
| E-9 행정 Workflow | E-9 제조업 HR 실무자, 외국인력 지원기관 실무자 | 업무 순서, 실제 담당자, 반복 여부, 완료 증빙 |
| 노무·급여·고용변동 | 공인노무사 또는 노무·급여 실무자 | 위험표현, 계산과 설명의 경계, 전문가 이관 조건 |
| 쉬운 한국어·다국어 안내 | E-9 교육 담당자, 이중언어 강사·검수자 | 이해하기 어려운 용어, 행동 지시의 명확성 |

인터뷰 참여자를 자동으로 전문가라 부르지 않습니다. 검수자의 역할, 관련 경험, 검수 범위가
해당 영역과 맞을 때만 전문가 검수로 기록합니다.

## 현실적인 검수 범위

### 100% 전문가 검수

- `high`, `critical` 민감도
- 두 팀원의 Intent·Domain·Workflow 불일치
- 법령, 필수서류, 기관 신고 절차를 정답으로 포함한 사례
- 급여 지연, 공제, 퇴사, 이탈 등 분쟁 가능 표현

### 층화표본 검수

- low·medium 사례는 Intent별 최소 20%
- 단일·복합·모호·범위 밖 사례를 각각 포함
- 같은 문장 구조가 반복되지 않도록 표현군 단위로 추출

표본의 수정·거절률이 10%를 넘으면 해당 Intent 또는 Workflow 전체를 재검수합니다.

## 검수 절차

```text
팀원 A·B 독립 라벨링
  -> 사전 일치도 계산
  -> 불일치 합의
  -> 고위험·표본 추출
  -> 전문가 O / 수정 / X 검수
  -> 라벨 가이드·Context Pack 반영
  -> GOLD_TEAM 또는 GOLD_EXPERT 승격
```

## 판정

| Decision | 의미 |
| --- | --- |
| `ACCEPT` | 실제 업무와 라벨이 적절함 |
| `ACCEPT_WITH_EDIT` | 수정 후 사용 가능 |
| `REJECT` | 실제 업무와 맞지 않거나 위험함 |
| `OUT_OF_SCOPE` | 해당 전문가의 검수 범위를 벗어남 |

## 기록 원칙

- 검수자의 실명은 동의 없이 저장하지 않고 `E9_HR_PRACTITIONER` 같은 역할 코드 사용
- 회사·근로자 개인정보와 원본 문서는 전달하지 않음
- 검수일, 대상 ID, 범위, 판정, 수정 요약, 근거 출처를 기록
- 전문가 의견이 공식자료와 충돌하면 공식기관에 재확인하고 근거를 함께 남김
- 전문가 1인의 의견을 전체 산업의 수요 근거로 일반화하지 않음

## MVP 최소 기준

1. E-9 행정 Workflow 검수자 1명
2. 제조업 한국어·다국어 안내 검수자 1명
3. 급여·고용변동을 MVP에 포함한다면 노무·급여 전문가 1명

세 번째 검수자를 확보하지 못하면 급여·고용변동은 자동 답변이 아니라 HR 검토·전문가 이관
Workflow로만 시연합니다.
