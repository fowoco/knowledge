# E-9 신고·연장 업무 기획 기준

## 한 줄 판단

인터뷰에서 확인된 신고 부담은 단순 알림 문제가 아니라, **사건 또는 만료일을 기준으로
필요 절차를 구분하고 앞 단계의 서류·접수 결과를 다음 단계에 연결하는 업무**다.

## 인터뷰 근거와 해석

- 외국인근로자 약 10명을 관리하는 사업장에서 신고 관련 업무를 가장 부담되는 일로 언급
- 담당자는 관련 업무를 합쳐 주 1~2시간 정도 수행하며, 10년 이상 경험해 절차에는 익숙함
- 고용센터에서 처리한 결과를 바탕으로 출입국 관련 업무를 이어서 준비하는 사례 언급
- 근로자별 시점이 달라 한 명의 업무가 끝나면 다른 근로자의 업무가 이어지는 점을 부담으로 설명

인터뷰의 “주기적 신고”는 법적 절차명이 아니다. 후속 확인에서는 실제 사례가 고용변동,
고용허가기간 연장, 취업활동기간 연장, 체류기간 연장 중 무엇이었는지 구분해 물어야 한다.

## 업무를 세 가지로 분리

| 구분 | 발생 기준 | 서비스 처리 | 외부 처리 |
| --- | --- | --- | --- |
| 근로계약·고용허가기간 연장 | 계약 만료일 | 의사 확인, 계약 준비, 서류·기한 카드 | HR이 EPS·고용센터 처리 |
| 체류기간 연장 | 체류 만료일 | 선행 결과·보유서류 확인, 다음 단계 연결 | HR 지원 아래 하이코리아·출입국 처리 |
| 고용변동 등 신고 | 퇴사·해고·사망·소재불명·중요 계약변경 사건 | 사건 후보 구조화, HR 사실확인, 기한 후보 계산 | HR이 공식 채널에서 처리 |

고용변동 신고는 동일 사실을 기본적으로 두 기관에 중복 제출하는 흐름으로 안내하지 않는다.
반면 계약·고용허가·체류 연장은 선행 결과가 필요한 다단계 사례가 있으므로 상태를 연결한다.

## FOWOCO가 실제로 하는 일

1. DB 만료일 또는 HR 입력에서 업무 후보 감지
2. 정확한 절차를 선택하고 애매하면 HR에게 후보 카드로 확인 요청
3. 공식 데이터에서 신청서별 필요서류 후보 조회
4. 보유 문서의 존재·유효기간·검증 상태 비교
5. 법정기한과 내부 준비일을 구분해 업무카드 생성
6. 근로자에게 받아야 할 서류만 다국어 요청 초안 생성
7. HR 승인 후 발송하고 제출·질문·미확인을 추적
8. 공식 사이트 링크 제공 후 HR이 직접 제출
9. 접수증 또는 미신고 사유를 증빙으로 등록하고 다음 Workflow 연결

## 기술 역할 분리

| 계층 | 책임 |
| --- | --- |
| Intent·Slot 모델 | “신고해줘” 문장에서 사건 후보·대상·날짜 추출 |
| Workflow Agent | 후보 사건을 절차로 연결하고 다음 단계 제안 |
| 규칙 엔진 | 확인된 기준일로 기한 후보 계산, 상태 전환, 증빙 누락 차단 |
| 공식 데이터 조회 | 신청서별 필요서류 후보와 출처 버전 반환 |
| HITL | 사건 유형·기준일·신고 필요 여부·최종 발송·제출 승인 |

AI가 신고 필요 여부를 최종 판단하거나 기관에 자동 제출하지 않는다. 특히 “무단결근”을
곧바로 법적 “소재불명”으로 확정하지 않고, 연락·근태 기록을 근거로 HR 확인을 요구한다.

## 대표 시나리오

```text
HR: "WRK-011이 어제부터 연락이 안 되고 출근하지 않았어"
  -> AI: 연락두절/결근 사건 후보, 대상·발생시점 추출
  -> HR: 사실관계와 마지막 확인시점 검토
  -> Workflow: 고용변동 신고 대상 여부 확인 카드 생성
  -> Rule: HR이 확정한 사건유형·기준일로 15일 기한 후보 계산
  -> Knowledge: 공식 근거·서식·필요서류 후보 표시
  -> HR: EPS·하이코리아 또는 관할 기관에서 직접 처리
  -> System: 접수증 또는 미신고 확인 메모로 완료
```

## 파일럿 검증 지표

성과를 미리 달성했다고 주장하지 않고 다음 기준을 측정한다.

| 지표 | 기준선·목표 |
| --- | --- |
| 주간 신고·연장 준비시간 | 인터뷰 기준 주 1~2시간, 파일럿 30% 감소 가설 |
| 절차 분류 정확도 | 전문가 검수 20사례 기준 90% 이상 목표 |
| 필요서류 검색 적합성 | 공식 신청서명 매핑 사례 Top-k Recall 95% 이상 목표 |
| 기한 계산 일치율 | 전문가 확정 기준일을 사용한 테스트 100% 목표 |
| 재요청·재방문 | 파일럿 전후 건수 기록, 감소 여부 측정 |

## 아직 해결하지 않은 항목

- 어떤 신고가 주 1~2시간의 대부분을 차지하는지 후속 인터뷰로 재분류
- 고용센터·출입국 순차 처리의 실제 사례명과 사용 서식 확인
- 관할 기관별 안내 차이가 있는지 담당자 또는 행정사·노무사 표본 검수
- `고용변동 등 신고` 고위험 사례 100% 전문가 검수
- 일반 사례 층화표본 20% 검수 후 오류율이 높은 유형 전체 재검수
