# 데이터 담당 후속 과업

## A. Seed 독립 검수

> 목표: 현재 40건의 제안 라벨을 정답으로 간주하지 않고 Intent·Domain 기준부터 안정화

- [ ] 검수자 A·B가 서로의 파일을 보지 않고 40건 작성
- [ ] Intent·Domain·Workflow·Slot·누락·모호성·다음 행동 입력
- [ ] `compare-reviews`로 Exact Match와 Macro Kappa 계산
- [ ] Kappa 0.70 미만인 라벨의 경계 사례와 가이드 수정
- [ ] 불일치 행만 합의표로 이동

**완료 조건:** 두 파일 40건 완료, 미완료 0건, 불일치 사유 기록, 라벨 가이드 개정

## B. 전문가 검수

> 목표: 팀이 분류 라벨은 만들 수 있어도 법정기한·필수서류·조치안을 임의로 확정하지 않도록 분리

- [ ] `high`·`critical` 중 법령·신고·체류·급여 조치안 전수 목록화
- [ ] 일반 사례는 Intent별 층화표본 20% 추출
- [ ] 전문가에게 Intent가 아니라 Workflow·필수정보·조치안 검토 요청
- [ ] 표본 수정률이 10%를 넘은 유형은 같은 유형 전체 재검수
- [ ] 팀 합의 라벨은 `GOLD_TEAM`, 전문가 보조라벨까지 통과하면 `GOLD_EXPERT`로 구분

**완료 조건:** 검수자·일자·수정 전후·근거 출처·전문가 범위 기록

## C. 분류 모델 데이터 확장

> 목표: 라벨 기준이 안정된 뒤에만 문체와 복합요청을 늘려 600건 Baseline 비교 가능 상태 확보

- [ ] Gold V1 최소 60건 확보
- [ ] 직접명령·질문·상황진술·짧은 메모·간접요청 문체 균형 확인
- [ ] 단일 요청뿐 아니라 복합요청·모호요청·범위 밖 요청 포함
- [ ] 원문 파생 문장은 `source_parent_id`로 묶어 같은 Split에 배치
- [ ] 원본 Gold를 먼저 Train·Validation·Test로 분리
- [ ] 증강 문장은 Train에만 추가하고 생성 모델·프롬프트 버전 기록

**완료 조건:** 총 600건, Intent별 최소 80건, PII·중복·Split 누수 0건

## D. Language Agent 실제 출력 평가

> 목표: 회귀평가용 의도적 오류 데이터와 별개로 실제 모델 후보의 생성 품질 측정

- [ ] 동일 Source Slot으로 후보 모델별 안내문 생성
- [ ] 생성문에서 Candidate Slot 재추출
- [ ] `check-notice-quality`로 핵심값 누락·변경·추가 검사
- [ ] 법령 단정·외부 실행·계산근거 부재를 별도 기록
- [ ] HR이 수정한 핵심값과 행동 수정률 기록
- [ ] 언어별 표본은 가능하면 해당 언어 사용자에게 행동 이해 여부 검수

**완료 조건:** 핵심값 보존율·치명값 누락률·근거 없는 값 추가율·HR 수정률 산출

## E. 파일럿 기준선

> 목표: 기대효과를 추정치가 아니라 도입 전후 같은 기준으로 측정

- [ ] `pilot_measurement_template.csv`를 파일럿 시작 전 주부터 작성
- [ ] 업무 건수와 전체 시간을 분리해 기록
- [ ] 재요청·반려·기관 재방문·외부 도움 요청 횟수 기록
- [ ] FOWOCO 적용 후 같은 항목을 다시 기록
- [ ] 감소율보다 원시 건수와 표본 기간을 함께 제시

**완료 조건:** 최소 2주 기준선과 2주 파일럿, 같은 Workflow의 전후 데이터 확보

## 병렬 진행 순서

```text
검수자 A/B: Seed 40건 독립 검수
데이터 담당: 불일치 분석·전문가 큐·데이터 리포트 유지
모델 담당: 잠긴 18건으로 후보 모델 Smoke Test
Agent 담당: 품질 검사기·마스킹 모듈 연동
기획 담당: 파일럿 기준선과 구매 질문 수집
```

데이터 담당자는 검수 완료 전에 600건부터 늘리지 않습니다. 현재 병목은 행 수가 아니라 라벨
경계와 고위험 조치안의 타당성입니다.
